Interface: IInstructionExecutionState

IInstructionExecutionState

The instruction function execution state keeps track of the progress of an instruction. An instruction ends when it returns truthy or it reached the end. The output value must be assigned to the out variable within the scope.
Properties:
Name Type Description
instructionName string An ordered instructions list.
nextFn int Next instruction within the instruction (list).
args Array.<any> The arguments to be passed to the next instruction function.
scope Object scope of the instruction set.
Properties
Name Type Description
out any The value returned should be assigned to this property. Everything will be accepted as an output variable. Even 'undefined'.
err Error The last error which occured. If the error is not assigned a value 'handled = true' the error will be passed to the the instruction will be ended if the error is not marked as handled. And then passed to the previous instruction and so forth.
